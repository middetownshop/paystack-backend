## Rule
The `src/components/ui/sonner.tsx` file was generated by shadcn with an `import { useTheme } from "next-themes"` dependency. This app has no theme switcher, so `next-themes` is not installed.

**Why:** shadcn generates sonner.tsx assuming next-themes for dark/light mode toggling. This project uses a fixed light theme.

**How to apply:** If sonner.tsx ever gets regenerated or causes a missing-module error, replace the `useTheme` logic with `theme="light"` hardcoded prop. Do not install `next-themes` — it adds unnecessary weight.
